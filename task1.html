GitHub Copilot in VS Code

01. Ghost Text & Comments-to-Code

Feature: Copilot Autocomplete
  • Concept: Real-time code suggestions based on natural language intent.
  • Mechanism: Copilot analyzes comments and surrounding code to predict the next lines.
Live Demo: Create a new file, type // Function to validate an email address, press Enter, and hit Tab.

02. Chat View & Slash Commands

Feature: Side Panel Chat
  • Commands: Use /explain to understand logic or /fix for errors.
  • Context: Use #file or #selection to provide specific background.
  • Flexibility: Switch between LLM models (GPT-4o / Claude) in the chat header.
Live Demo: Open Ctrl+Alt+I, type #file:package.json Summarize the dependencies.

03. Agent Mode (Copilot Edits)

Feature: Copilot Edits
  • Agentic Behavior: Can plan and execute changes across multiple files simultaneously.
  • Goal: High-level feature implementation (e.g., adding a new API route and a UI component).
Live Demo: Open Edits (Ctrl+Shift+I), prompt: "Add a basic Contact form component and a route to handle it."

04. Autonomous Testing & Fixing

Feature: Terminal & Agent Integration
  • Generation: Automatically creates unit tests for existing logic.
  • Self-Healing: Agent can run the test, read terminal errors, and fix the code autonomously.
Live Demo: In Edits, ask: "/tests create unit tests, run them, and fix if they fail."

05. Inline Chat (Quick Actions)

Feature: Inline Chat (Ctrl+I)
  • Focus: Edit code without switching to a side window.
  • Refactoring: Best for adding try/catch, documentation, or CSS classes.
Live Demo: Highlight a function, press Ctrl+I, type "/doc wrap this in try-catch and add JSDoc."

06. Custom Instructions

Feature: .github/copilot-instructions.md
  • Persistence: Define project-wide rules (e.g., "Use TypeScript", "Use Tailwind").
  • Automation: Copilot automatically includes this file in every prompt context.
Live Demo: Create the file .github/copilot-instructions.md and define a specific coding style.
