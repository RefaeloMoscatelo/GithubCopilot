GitHub Copilot

GITHUB COPILOT TRAINING

בהנחה שה GITHUB COPILOT כבר מותקן אצל המתכנתים\ הQA\ וכבר עברו הדרכה

שגרה יומית

  1. משימות – אנו כמדריכים מקבלים משימות לעזרה\ בעיה עם ה COPILOT שחברי הצוות לא הצליחו לפתור וצריכים את עזרתנו, יום כזה גם יכול לכלול DAILY שבו נעבור על אתגרים\ קשיים שהיו לחלק מחברי הצוות ואיך לפתור אותן וכך גם כולם ילמדו מטעויות של הקולגות.
  2. שיעור תרגול – באמצע היום יהיה שיעור של שעה על פיצ'ר חדש ותרגול שלו או הסבר על ייעול הטקסט מול ה GITHUB COPILOT
  3. Copilot Code Reviews – מעבר בין הצוותים ובדיקה של הקוד, האם ה GITHUB COPILOT עובד כנדרש ולשביעות רצונם, לבדוק מה אפשר לשפר, לקבל חיווי איפה הנקודות החלשות שצריכות הסבר, כאן גם אפשר לבצע בדיקות כגון האם ישנם פיצ'רים שבהם לא נעשה שימוש בכלל.

שגרה שבועית

  1. Show & Tell שבועי - דיון פתוח שבו אפשר לתת למפתח להציג משהו שאותו גילה לטובה, פיצ'ר שקידם את העבודה שלו משמעותית למשל: כתיבת Unit Tests מורכבים ב-5 דקות.
  2. הפקת לקחים - נחזור על אתגרים שליוו אותנו במשך השבוע ומה למדנו מהטעויות, מהבעיות והפתרונות שלהם ומהפיצ'רים החדשים.
    • Prompt Engineering Clinic - סקירת מקרי בוחן שבהם ה-Copilot "זייף" והסבר איך לדייק את ההנחיה.
  3. חדשות ועדכונים – פיצ'רים חדשים שיצאו, עדכוני גרסאות, מה אנחנו כצוות מטמיעים ומדריכים למדנו מהליווי והעזרה ומה המסקנות שלנו ממה שראינו מהעבודה שלהם עם ה GITHUB COPILOT

שגרה חודשית - "מבט על ושיפור ביצועים"

  1. ניתוח דאשבורד (Usage Reports) – נבדוק כל צוות מה היה אחוז ה-Acceptance Rate וצריכה לפי שפות תכנות.
  2. פגישה עם מנהלי צוותים בלבד – קבלת פידבק, בדיקה מה אפשר לשפר מבחינת התמיכה שלנו בצוותים.
  3. AI Champion Awards - הוקרה למפתחים שהובילו אימוץ בצוות שלהם (Gamification).

הערות כלליות

  • בכל דיון נושא אבטחת המידע יכול לעלות גם מצידנו וגם מצד המתכנתים

כלים / עזרים

כלי / עזר שימוש מרכזי
GitHub Copilot Dashboard מעקב אחרי אחוז הקוד שאומץ וחיסכון בזמן (Productivity Metrics).
Internal Wiki (Confluence) מאגר "מתכוני" Prompting מותאמים לשפות התכנות הפנים-בנקאיות.
Custom Extensions פיתוח תוספים שמחברים את Copilot ל-API הפנימי של הבנק.
Surveys (Qualitative) סקר חודשי למפתחים: "האם ה-AI הפחית את העומס הקוגניטיבי שלך?".

היבטי מדידה ואימפקט (KPIs)

  • מהירות (Velocity): צמצום זמן ה-Cycle Time במשימות Coding סטנדרטיות.
  • איכות (Quality): ירידה בכמות הבאגים ב-Unit Tests (בזכות כיסוי בדיקות רחב יותר).
  • שביעות רצון (Developer Happiness): מדד ה-eNPS של המפתחים כלפי סביבת העבודה.
  • אפשר גם להשוות כמות משימות שנעשו לפני הטמעת ה GITHUB COPILOT לעומת אחרי.
